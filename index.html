Zenponsive - A responsive framework draft
with (Zenphoto) gallery themes in mind

Admittedly this is not that different from lots of other grid frameworks and probably a bit less sophisticated. It was made to be a simple grid that has no predefined design styles. The features:

  1. Fluid scaling percent based layout

    ca. 960px base width (can be adjusted generally) and only two mediaquery breakpoints of 767px and 479px.

  2. No gutter

    That way you can easily combine or nest columns around without needing to care about different left, middle or right margins for the first or last column in a row. You just set any margin/padding using your elements within the column using an custom css file. More freedom to you.

    (note due percentage rounding differences the grid is not exact and the same on all browsers).

  3. No predefined styles and typography

    All styling up to you. Only a normalize css to eliminate browser differences is included

  4. Only two default mediaqueries

    Only 767px and 479px are included since the framework is fully fluid. You can easily extend if you need or want more via your custom css file.

  5. Extra column classes

    By default all columns becomme full width (col12) on the smallest view. But there are extra ones for some columns to force different behaviour:

    • .xxx-479/.xxx-767/.xxx-960: For columns directly
    • .xxx-479-n/.xxx-767-n/..xxx-960-n: For columns nested within other columns.

Browser support

Generally tested with latest Safari, Firefox, Opera, Chrome. Base layout also seems to work in IE7+ (responsiveness not tested)

Because of the percentage roundings is different in all browsers it may not always be 100% accurat

Overriding behaviour using the extra classes

You can use the special classes mentioned above to override the behaviour. SO you can choose to turn colums into different ones on the smaller resolutions.

.col01 wrapped in .col02-767-n and .col04-479-n

Nested columns

You can nested classes of the size col01(-extra) to col06(-extra) within col03-10. It does not make sense to nested within col01 or col012. The first is way too narrow and for the latter you better can just use columns directly.

The nested cols must be numbered with the classes .one to .six to eliminate the left/right borders of the first and last in the row.

Besides that you can put several rows within without any extra wrapper.
